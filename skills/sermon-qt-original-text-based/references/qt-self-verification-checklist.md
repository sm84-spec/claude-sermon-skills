# 큐티 자기 검증 체크리스트 (C1–C8) — 운용 매뉴얼

SKILL.md의 *자기 검증 체크리스트* 절을 보강하는 매뉴얼. 각 항목을 산출물 출력 직전에 점검하고, 통과하지 못하는 항목은 재작성 후 사용자에게 제시한다.

## C1. 본문 인용 정확성

### 점검 사항
- [ ] 인용한 모든 개역개정 본문을 정확히 옮겼는가? (토씨·조사 단위까지)
- [ ] 새번역·NIV·KJV·NASB·표준새번역의 표현이 섞이지 않았는가?
- [ ] 장절 번호가 사용자가 준 범위와 정확히 일치하는가?
- [ ] 사용자가 명시하지 않은 절을 확장 인용하지 않았는가?

### 자주 발생하는 오류
- "사망의 음침한 골짜기" (개역개정) vs "사망의 그늘진 골짜기" (새번역) 혼동
- "능력 주시는 자 안에서" (개역개정) vs "내게 능력 주시는 분 안에서" (새번역) 혼동
- "오직 너희는 택하신 족속이요" (개역개정 벧전 2:9) — 토씨 다 외워 단언 위험
- 장절 표기 실수: "롬 8:28–30"으로 인용 후 본문은 28–29절만 다룬다거나, 사용자가 28절만 요청했는데 임의 확장

### 처리
- 잘 알려진 핵심 구절(요 3:16, 시 23:1, 마 5:3–10 등)만 직접 인용을 시도.
- 잘 알려지지 않은 구절은 첫·끝 절만 직접 인용하고 사이는 *"[해당 절: 개역개정 본문을 직접 참조해 주시기 바랍니다]"*로 명시.
- 산출 끝에 짧게 *"본문 인용은 개역개정판으로 직접 확인하시기 바랍니다"*를 안내해도 좋다.

## C2. 원어 정확성

### 점검 사항
- [ ] 헬라어 철자(spirit marks·iota subscript·accent)가 정확한가?
- [ ] 히브리어 자음·모음점(특히 다게쉬·마터르 레크티오니스)이 정확한가?
- [ ] 영어 음역이 SBL Handbook of Style 2판 규약과 일치하는가?
- [ ] 한글 발음 표기가 `original-language-transliteration-standard.md`의 표에 부합하는가?
- [ ] 음역과 한글 발음 표기가 *서로 일관되는가*? (예: 음역 "dialegomai"인데 한글이 "다일레고마이"라면 불일치)

### 자주 발생하는 오류
- η를 한글로 "이"로 (modern Greek 발음, Erasmian 위반)
- θ를 한글로 "쓰"로 (modern Greek 발음, Erasmian 위반)
- ου를 한글로 "오우"로 (오우는 영어 표기, 올바른 발음은 "우")
- 거친 숨표 ̔를 누락하여 ὁ를 "오"로 (정확: "호")
- 히브리어 ḥ(헤트)를 단순히 "ㅎ" 대신 "흐"로 옮겨 어색
- 다게쉬 부재로 베트(ב)와 베트(בּ)의 한글 표기를 혼동
- 시제·법·태 단언: "이 동사는 미완료 직설법"이라 단언했는데 실제 본문에서는 부정과거 직설법인 경우

### 처리
- 단어가 본문 그 자체에서 어떤 형태로 등장하는지 확실하지 않으면 *어휘형(lexical form)*만 제시하고 본문 형태는 짧게 ("본문에서는 활용형으로 등장") 또는 생략.
- 활용형 단언은 본문에서 명확히 확인 가능한 단어에 한정.

## C3. 원어와 본문의 일치

### 점검 사항
- [ ] 인용한 원어 단어가 사용자가 지정한 본문(또는 시스템이 결정한 본문)에 실제로 등장하는가?
- [ ] "이 본문의 핵심어"라고 제시한 단어가 정말 그 본문에 있는가?

### 자주 발생하는 오류
- 본문에 없는 단어를 자기 신학 체계에서 끌어오기 (예: 빌 4:4–7에서 다루어지지 않는 "δικαιοσύνη"를 핵심어로 제시)
- 본문의 한국어 단어와 헬라어/히브리어가 일대일 대응한다고 가정 (개역개정 번역은 어휘 일대일 대응이 아닌 경우 다수)
- 본문에 ἀγάπη가 등장한다고 단언했는데 실제 본문은 ἀγαπάω(동사형)인 경우

### 처리
- 원어 단어 선정 시 본문 안에 실제 등장하는 형태를 확인.
- 본문 안의 어떤 한국어 단어가 그 원어로 옮겨졌는지 짧게 안내 (예: "본문 7절 '관용' = ἐπιεικές").

## C4. 3대지 구조

### 점검 사항
- [ ] 출력이 정확히 3대지로 구성되었는가? (2대지 또는 4대지 아님)
- [ ] 그 중 한 대지가 *대지3 구조 규약*에 따라 "원문 추출 3가지 의미"인가?
- [ ] 기본형(한 단어의 3가지 의미 층) / 대안형(3개 다른 핵심 단어) 중 어느 것인지 대지 제목 또는 첫 줄에 명시했는가?
- [ ] 3가지 의미가 서로 구별되는가? (사실상 같은 의미의 반복이 아닌가?)

### 자주 발생하는 오류
- 대지가 본문 흐름이 아닌 추상적 주제 3개로 추상화 (예: "은혜·믿음·소망"으로 단순화)
- 대지3의 3가지 의미가 사전적 다의 전체를 본문 함의 없이 나열
- 대지 제목이 너무 길거나 본문과 무관한 캐치프레이즈

### 처리
- 대지 1·2는 본문 흐름(절의 순서)을 반영. 예: 대지1 (8–12절), 대지2 (13–17절).
- 대지3은 원어 분석으로 본문 전체를 다시 한번 종합.

## C5. 본문 비약 차단

### 점검 사항
- [ ] 적용·묵상 질문·기도가 본문이 *실제로 말하는* 범위 안에 있는가?
- [ ] 자주 비약되는 본문(빌 4:13·렘 29:11·마 18:20 등)을 본문 문맥대로 풀었는가?
- [ ] 번영신학("이 본문을 따르면 부와 성공이 약속된다") 풀이가 끼어들지 않았는가?
- [ ] 청중에게 듣기 좋은 일반론(인생 동기부여)으로 본문을 무화하지 않았는가?
- [ ] 책망 본문을 위로로, 위로 본문을 책망으로 변색하지 않았는가?

### 자주 발생하는 오류
- 빌 4:13을 "내가 무엇이든 다 해낼 수 있다는 자기계발 약속"으로 풀이
- 렘 29:11을 개인 출세·사업 성공의 약속으로 풀이
- 마 18:20을 모임 인원수 부족 합리화 본문으로 풀이
- 사 41:10·43:1을 개인 위로 본문으로만 풀이(원래 이스라엘 공동체 약속)
- 시 1편의 "복 있는 사람"을 일반 행복론으로 풀이
- 마 7:7("구하라")를 모든 기도가 응답된다는 만능 약속으로 풀이
- 슥 4:6("힘으로 되지 아니하며")을 개인 노력 부정의 본문으로 풀이

### 처리
- 본문 직전·직후 문맥(최소 1단락)을 머릿속으로 확인.
- 적용은 본문 안의 명령·약속·교훈에서 직접 도출.

## C6. 출처·인물·통계 단언 차단

### 점검 사항
- [ ] 학자 이름·논문·출판사·연도를 단언하지 않았는가?
- [ ] 통계("○○년 ○○ 통계", "한국인의 ○○%")를 단언하지 않았는가?
- [ ] 역사 인물의 일화를 자세한 검증 없이 단언하지 않았는가?
- [ ] 영화·도서·시의 줄거리·구절을 자세히 모르는데 단언하지 않았는가?
- [ ] "○○ 박사에 따르면", "○○ 교수가 말하길"을 출처 없이 사용하지 않았는가?

### 자주 발생하는 오류
- "BDAG에 따르면 이 단어는…" — BDAG에 있는지 확신 없는 풀이
- "한 통계에 따르면 한국 그리스도인의 70%가…" — 출처 없는 수치
- "마틴 루터가 '커피 한 잔' 어쩌고…" — 출처 불명 일화
- "톨스토이의 단편 ○○에서는…" — 줄거리 단언이 부정확

### 처리
- 일반화 표현으로 전환: "주류 사전은", "한국 교회에 자주 알려진 일화는", "정확한 통계는 확인 필요하나 통상…"
- 확신 없는 인용은 통째로 삭제.

## C7. 분량과 형식

### 점검 사항
- [ ] 사용자가 지정한 레벨(또는 기본값 표준)의 분량 범위에 맞는가? (간단 800–1,500자 / 표준 2,000–3,500자 / 상세 4,000–6,500자, 한글 음절 기준)
- [ ] 시작 헤더 형식(`# 📖 [본문 약어] 큐티 — [한 줄 주제]`)을 지켰는가?
- [ ] 레벨별 구조 템플릿의 모든 절을 포함했는가?
- [ ] 묵상 질문이 각 대지 끝에 한 줄로 들어갔는가?

### 자주 발생하는 오류
- 표준 QT라고 명시했는데 실제 분량이 2,000자 미만 또는 3,500자 초과
- 헤더에 약어 형식 비준수 (예: "사도행전 19장 8-20절" 대신 "행 19:8–20"이 표준)
- 묵상 질문 누락
- 기도 항목 누락

### 처리
- 산출 후 단어 수 가늠. 지정 범위 벗어나면 늘리거나 줄이기.
- 모든 템플릿 절 점검 (오늘의 말씀 → 서론 → 원문 분석 → 대지1·2·3 → 적용 → 기도).

## C8. 검색 출처 투명성

### 점검 사항
- [ ] 웹 검색을 사용했으면 짧은 출처를 표기했는가?
- [ ] 사용하지 않았으면 한 줄 투명성 안내가 들어갔는가?
- [ ] 검색이 부적합·빈약했을 때 그 사실을 사용자에게 알렸는가?

### 자주 발생하는 오류
- 검색하지 않고도 "한 신문 기사에 따르면…"이라고 출처 없이 단언
- 웹 검색 결과가 빈약했는데 마치 충실히 검색한 듯 작성
- 일반 예화를 사용하면서 마치 최근 기사인 듯 단언

### 처리
- 검색 미사용 시 산출 첫머리에 *"본 응답은 외부 자료 검색을 사용하지 않았으며, 서론 예화는 일반 비유로 대체했습니다"* 한 줄.
- 검색 사용 시 짧은 출처 표기.

## 통과 후 처리

- C1–C8 모두 통과 → 사용자에게 산출물 제시.
- 한 항목이라도 미통과 → 그 부분을 재작성한 후 다시 점검. 두 번 재작성으로도 통과 못 하면 사용자에게 *"본 본문은 ○○ 사유로 일반 산출에 제약이 있어, 부분적으로 보완 인용으로 처리했습니다"*라고 짧게 알린다.

## 노출 규약

체크리스트 본문은 응답에 노출하지 않는다. 사용자가 "자체 검증 결과도 보여줘"라고 명시 요청하면 응답 말미에 8개 항목 체크 표만 짧게 제시한다 (✓/△/✗).
