# Multi-Bible Version Compare — 작업 방법론

본 스킬은 한 본문을 9개 layer로 펼쳐 보여주는 *고밀도 비교* 도구다. 본 방법론 문서는 작업의 단계·결정 기준·자기 검증 절차를 명시한다.

---

## 1. 작업 단계 (Workflow)

### 단계 1 — 입력 분석
- 본문 식별: 신약 / 구약 / 시편 / 잠언 / 율법 / 예언서 / 외경(천주교 포함시) / 그 외
- 본문 분량: 1절 / 2-5절 / 6절 이상 / 장 단위
- 본문 유명도: 매우 유명 (요 3:16·시 23 등) / 보통 / 흔하지 않음
- *질문에 모호함이 있다면* 단 한 번만 확인.

### 단계 2 — references/ 로드 (강제)
출력 시작 전에 다음 references를 모두 검토:
1. `anti-hallucination-checklist.md` — 9개 layer 인용 점검표
2. `lxx-mt-numbering.md` — 시편·예레미야·잠언 번호 차이
3. `quran-verified-citations.md` — 코란 검증된 인용
4. `talmud-verified-citations.md` — 탈무드 검증된 인용
5. `vulgata-facts.md` — Vulgata 판본 차이
6. `common-misreadings.md` — 흔한 오류 카탈로그

### 단계 3 — scripts/ 활용
필요 시 결정론적 도구로 인용 사실 확인:
- `lookup_psalm_numbering.py [MT 번호]` → LXX·Vulgata 번호 반환
- `lookup_jeremiah_numbering.py [MT 장:절]` → LXX 매핑 반환
- `verify_citation_format.py [본문]` → 코란/탈무드/Vulgata 인용 형식 검사
- `verify_output_structure.py [본문]` → 9개 layer + 분석 섹션 모두 존재 검사
- `verify_caritas.py [본문]` → caritas vs cardinal virtue 혼동 검사

### 단계 4 — 9 layer 본문 수집
각 layer에 대해:
- **확신 인용 가능** → 정확 본문
- **확신 부족** → "[자신 없음 — 사용자 확인 필요]" 명시
- **해당 없음** (예: 신약 본문의 히브리어 원문) → "해당 없음 (구약 인용 없음)" 표기
- **직접 대응 없음** (코란·탈무드) → "직접 대응 본문 없음. 유사 주제 언급: ..."

### 단계 5 — 분석 작성
- 차이점 (A·B·C·D 4섹션)
- 공통점 (1섹션)
- 번역 포인트 (각 layer별 1단락)
- 종합 통찰 (2-4문장)

### 단계 6 — 자가 검증
- anti-hallucination-checklist.md 모든 점검 항목 통과 확인
- common-misreadings.md 30+ 오류 카탈로그 자가 점검
- 출력 직전 1회 더 점검

### 단계 7 — 출력 + 후속 안내

## 2. 결정 기준 (Decision Rules)

### 추가 번역(7번 slot) 선정
- **시편/잠언/욥기 등 시문학**: 새번역 또는 ESV (의미 명료성)
- **율법/역사서**: NASB 또는 KJV (단어 보존)
- **신약 서신**: NASB 또는 NIV 2011 (정확성)
- **공관복음**: 새번역 또는 NLT (서사 자연스러움)
- **요한복음**: ESV (신학적 정밀)
- **사용자가 명시한 번역**: 그것 우선

### 코란 본문 8번 slot
- **직접 대응 본문 있음** (예: 모세·아브라함·예수 사건): 인용
- **간접 대응** (인물·교훈만): "직접 본문 없으나 인물 ~ 언급"
- **대응 전무** (시편·서신 등 기독교 고유 본문): "코란에 직접 대응 없음. 이슬람의 일반적 입장만 언급"

### 탈무드 본문 9번 slot
- **율법/창세기**: 풍부한 대응 자료 (b. Sanhedrin·미드라쉬)
- **시편·잠언**: 미드라쉬 시편 또는 미쉬나 아보트
- **신약 본문**: 매우 제한적. "탈무드에는 직접 대응 없음. 일부 인물·주제에 대해 ~ 정도 언급"
- **메시아 본문**: 두 메시아 전통(메시아 벤 요셉·메시아 벤 다윗) 언급 가능
- **예수 본문**: b. Sanhedrin 43a 등 매우 제한된 언급. 인용 자제 권장.

## 3. 자기 검증 절차 (Self-Verification)

### 인용 작성 시
매 layer 인용 후 다음 자문:
- "이 본문을 나는 정말 정확히 외우고 있는가?"
- "본문 외운 척이 아닌가?"
- 자신 없으면 즉시 `[자신 없음 — 사용자 확인 필요]`로 전환

### 출처 검토 시
- 코란 수라:절 번호 — 일반 표준판(Cairo 1924)인가?
- 탈무드 b./y. 마쎄켓 페이지 — Vilna판 또는 Steinsaltz판 기준인가?
- Vulgata — Clementine·Nova Vulgata·Stuttgart 중 어느 것인가?
- LXX 시편 번호 — MT와 1 차이 매핑 확인했는가?

### 신학적 안전장치
- 혼합주의 결론으로 가지 않았는가?
- 반유대주의·반이슬람주의로 흐르지 않았는가?
- 정통 기독교 신학을 양보하지 않았는가?
- 번역 절대화하지 않았는가?

## 4. 모호함의 처리

### 모호한 본문 입력
- 단 1회만 확인: "어느 본문을 비교할까요? 예: ..."
- 사용자가 동일 모호 응답 시: 가장 표준적 본문 선정 + 그 사실 명시

### 인용 자체 불확실
- 절대 추정 인용 금지
- `[자신 없음 — Bible Gateway·Sefaria·STEPbible 등 확인 권장]` 표기

### 코란·탈무드 대응 자체 불확실
- "이 본문에 대한 코란/탈무드의 직접 언급은 확인되지 않음. 다만 ~ 정도가 관련될 수 있음"
